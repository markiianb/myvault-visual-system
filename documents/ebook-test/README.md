# Ebook test — "The Architecture of Trust"

## What this is

The test fixture for the MyVault ebook pipeline. Validates that `chunks/ebook.md` v1.1 produces a brand-correct PDF when rendered through Typst with the canonical template. **This is not production fundraising or marketing copy** — it is a real-shaped editorial piece written specifically to stress-test the design system end-to-end.

## What it validated

- **All 7 HARD rules** — R-EBOOK-001 (page geometry + 1-col / 2-col grid), R-EBOOK-002 (surface by category + Icon-left/page-number-right footer with surface-flip override), R-EBOOK-003 (signal-go 60×2 rule on Title Page + TOC), R-EBOOK-004 (every cover has a lockup), R-EBOOK-005 (mandatory page structure), R-EBOOK-006 (reading order), R-EBOOK-007 (signal-go endnote numbers).
- **11 distinct MENU page types** — Cover (T1), Title Page, Copyright, TOC, Section Opener (×3), Body 1-col, Pull Quote, Stat V1, Body 2-col, Stat V3 mirror on Teal, Callouts, Lists, Conclusion, Endnotes, Back Cover.
- **Foundation-rule inheritance** — R-COLOR (every surface and text color resolved from tokens), R-TYPE (PT Serif Regular + Lato Regular only, no bold/medium/italic), R-LOGO (Icon variants flip per pairings), R-ICON (signal-go markers on lists + endnotes).

## Pipeline

```

              ↓
```

## Iteration history

| Cycle | What | Outcome |
|---|---|---|
| 0 (smoke) | Placeholder-content spec, verify template compiles | 13 pages rendered. Found: body heading used Typst default font (not PT Serif); stat-mirror labels splayed apart from numbers; cover title not balanced. |
| 1 | Fix template: PT Serif heading override, cover layout balance, stat-mirror alignment. Add lists / callouts / 2-col / endnotes functions. | Smoke test re-rendered, fixes confirmed. |
| 2 | Real-content spec written ("The Architecture of Trust"). 19-page render. | Found: phantom blank page after cover (overflow); `#super[N]` rendered as literal text; cover title at 64pt wrapped awkwardly. |
| 3 | Reduce cover blocks to 520pt to suppress overflow; change `"..."` → `[...]` for content with markup; cover title default 64pt → 56pt. | 18 pages, no phantom, superscripts render as ² ³ ⁴, cover title fits cleanly on one line. **Done.** |

## Findings for the chunk doc

These observations may inform a future v1.2 of `chunks/ebook.md`:

1. **Cover title size belongs in BASE.** Current chunk doesn't specify a default; v1.1 of the template uses 56pt. Worth adding to BASE typography table.
2. **Body whitespace is normal.** When body content is shorter than a page, trailing whitespace is editorial-correct. Worth a one-line note in BASE so authors don't try to over-fill pages.
3. **The cover composition split height is sensitive to overflow.** Authors using exact 540+40+540=1120pt will hit a phantom-page bug. Recommend chunk note: "leave 20–40pt of slack inside the 1120pt content area."
4. **Renderer note: pass content blocks (`[...]`), not strings (`"..."`), for any field that includes Typst markup like superscript or footnote markers.** Worth adding to the renderer section.
